# Contexto

Se requiere una arquitectura simple,
modular y fácil de mantener.

# Decisión

El proyecto se organizará por módulos
de dominio.

Cada módulo tendrá:

- api
- application
- domain
- infrastructure

Los casos de uso se organizarán
en carpetas independientes.

# Consecuencias

+ Mayor mantenibilidad.

+ Casos de uso aislados.

- Más archivos.
